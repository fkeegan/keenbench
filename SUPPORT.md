# Support

## Getting Help

- Usage questions and troubleshooting: open a GitHub Discussion or Issue.
- Bugs: open an issue using the Bug Report template.
- Security vulnerabilities: follow `SECURITY.md` and do not file public issues.

## What to Include

When requesting support, include:

- OS and environment (`linux`/`macOS`, Flutter/Go versions)
- Exact steps to reproduce
- Expected behavior and actual behavior
- Relevant logs or screenshots (with secrets removed)

## Response Model

This is currently maintainer-led OSS support.

- Best-effort response for questions/issues
- No guaranteed SLA for general support
- Security reports follow the timeline in `SECURITY.md`
